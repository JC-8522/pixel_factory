# Generated Reference Usage

## Purpose

This note explains how the generated reference art should be used by product, frontend, and future art production.

## Generated Reference Set

The first-pass generated reference board covers:

1. empty office floor shell
2. workstation state treatment
3. occupied workstation with seated agent
4. modal and chat drawer UI skin

## How To Use It

### Product / Design

Use it as the visual truth source for:

- room composition
- workstation density
- hover and selection intensity
- panel tone and button color hierarchy

### Frontend

Use it to derive:

- scene layer order
- workstation box size
- label position
- chat drawer width and padding
- modal visual proportions

### Art Production

Use it as the concept target for:

- slicing modular workstation assets
- painting the final room shell
- producing agent sprite variants
- building reusable UI panel skins

## Important Limitation

The generated board is a style and composition reference, not the final production asset pack.

Production assets should still be exported as:

- transparent PNG components
- sprite sheets
- tiles
- 9-slice UI panel assets

## Recommended Next Art Step

Create the first production-ready asset batch in this order:

1. floor room shell
2. workstation base and overlays
3. seated agent sprite variants
4. modal and chat panel skins
